- create-next-app@15 (NOT @latest — that installs Next.js 16 with breaking async API changes)

- React Bits: NOT an npm package — install via shadcn CLI registry per component

## Scaffold flags (locked in CONTEXT.md)
npx create-next-app@15 . --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"

## Resulting directory layout
src/
  app/          (Next.js App Router)
  components/ui/  (shadcn-generated primitives)
  lib/utils.ts    (shadcn cn() utility)

## React Bits install pattern
npx shadcn@latest add https://reactbits.dev/registry/blur-text-TS-TW.json
NOTE: verify the exact registry URL at reactbits.dev/text-animations/blur-text before running.
The component arrives at src/components/ui/blur-text.tsx (or similar path).

## Code style (CLAUDE.md Section 4.3)
- Named exports: export function Foo() — NOT export default function Foo()
- No any types
- interface for objects, type for unions
- PascalCase component files

Answer YES — .planning/ and AGENT_PLAN.md are not Next.js files and will not be touched.
