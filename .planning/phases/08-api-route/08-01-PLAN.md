```typescript
const MAX_HTML_SIZE = 10 * 1024 * 1024;  // 10 MB in bytes
const MAX_MEMO_SIZE = 25 * 1024 * 1024;  // 25 MB in bytes
const VALID_MEMO_EXTS = ['.pdf', '.docx', '.md'] as const;
```

**Helper: sanitizeName (private, not exported):**
- Lowercase, spaces → hyphens, strip all non-alphanumeric except hyphens
- Edge case: if result is empty string (e.g., all special characters), fall back to 'participant'
```typescript
function sanitizeName(name: string): string {
  const sanitized = name
    .toLowerCase()
    .replace(/\s+/g, '-')
    .replace(/[^a-z0-9-]/g, '');
  return sanitized || 'participant';
}
```

**Helper: getExtension (private, not exported):**
- Use lastIndexOf('.') NOT path.extname — avoids importing the path module and handles filenames with multiple dots
- Returns lowercase extension including the dot: '.html', '.pdf', '.docx', '.md', or '' if no dot
```typescript
function getExtension(filename: string): string {
  const lastDot = filename.lastIndexOf('.');
  return lastDot === -1 ? '' : filename.slice(lastDot).toLowerCase();
}
```

**Export: POST handler:**
```typescript
export async function POST(request: NextRequest): Promise<NextResponse> {
```

Implementation steps inside POST (in this order):

1. Parse: `const formData = await request.formData();`

2. Extract all four values:
   ```typescript
   const name = formData.get('name');
   const email = formData.get('email');
   const htmlFile = formData.get('htmlFile');
   const memoFile = formData.get('memoFile');
   ```

3. Validate fields FIRST (return 400 immediately on first failure):
   - `name` must be a non-empty string after trim
   - `email` must be a non-empty string after trim
   - Use presence check only for email (NOT regex) — email is for organizer reference only; aggressive regex rejects valid addresses
   - Error message format: 'Participant name is required', 'Email address is required'

4. Validate files AFTER fields (type guard with instanceof File, then extension, then size):
   - htmlFile: instanceof File check → error 'HTML dashboard file is required'
   - htmlFile extension: getExtension(htmlFile.name) === '.html' → error 'HTML file must be .html format'
   - htmlFile size: htmlFile.size > MAX_HTML_SIZE → error 'HTML file exceeds the 10 MB limit'
   - memoFile: instanceof File check → error 'Findings memo file is required'
   - memoFile extension: VALID_MEMO_EXTS.includes(memoExt) check → error 'Memo file must be .pdf, .docx, or .md format'
   - memoFile size: memoFile.size > MAX_MEMO_SIZE → error 'Memo file exceeds the 25 MB limit'
   - All 400 responses shape: `NextResponse.json({ success: false, error: '...' }, { status: 400 })`

5. Build Blob path prefix:
   ```typescript
   const sanitized = sanitizeName(name.trim());
   const timestamp = Date.now();
   const prefix = `submissions/${sanitized}-${timestamp}`;
   const memoExt = getExtension(memoFile.name);
   ```

6. Upload both files inside try/catch:
   - Use Promise.all for parallel upload (faster; acceptable to have orphaned blob on partial failure for this competition use case)
   - Dashboard path: `${prefix}/dashboard.html`
   - Memo path: `${prefix}/memo${memoExt}` (e.g., memo.pdf, memo.docx, memo.md)
   - Both use `{ access: 'public' }` — required field; do NOT add `addRandomSuffix: true` (timestamp provides uniqueness)
   - On success: return `NextResponse.json({ success: true, submittedAt: new Date().toISOString(), files: { dashboard: dashboardBlob.url, memo: memoBlob.url } })`
   - On catch: return `NextResponse.json({ success: false, error: 'File upload failed. Please try again.' }, { status: 500 })`

The BLOB_READ_WRITE_TOKEN is read automatically by the @vercel/blob SDK from process.env — do NOT pass it explicitly. Do NOT reference process.env.BLOB_READ_WRITE_TOKEN anywhere in this file (the SDK handles it internally).

The final route.ts file must have no 'use client' directive, no default export, and no imports from any file in src/ (it is fully self-contained).
